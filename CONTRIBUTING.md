# Contributing

Thanks for helping maintain this reading list.

## Inclusion Criteria

A paper is in scope when data is central to the contribution, not only mentioned as implementation detail. Good fits include:

- Data collection, curation, filtering, deduplication, mixing, curricula, or scaling laws.
- Data attribution, influence, valuation, markets, provenance, licensing, copyright, privacy, or unlearning.
- Synthetic data, recursive training, model collapse, or data generated by foundation models.
- Benchmark contamination, freshness, dynamic evaluation, RAG data, or contamination detection.
- Safety/fairness/alignment work where the main artifact is a dataset, annotation protocol, or data filtering method.
- Multimodal data pipelines, data mixture studies, or data attribution for vision-language/diffusion models.

## Entry Format

Use this Markdown format:

```markdown
- [YYYY] [Paper Title](https://paper-link) - Short data-centric note.
```

Prefer `arxiv.org/abs/...`, OpenReview, ACL Anthology, USENIX, official project pages, or publisher pages. Avoid only linking to news coverage unless no paper/project page exists.

## Updating the YAML Index

When adding a paper, also update `data/papers.yaml` if possible:

```yaml
- year: 2026
  title: "Paper title"
  url: "https://arxiv.org/abs/..."
  category: "pretraining-data"
  tags: ["data-selection", "mixture"]
```

Keep categories broad and stable. Add new categories only when the existing taxonomy clearly does not fit.
